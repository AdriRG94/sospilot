SOSPilot

Home of the RIVM/Geonovum SOSPilot project

The main goal of this project is to take raw Air Quality Measurements from the Dutch RIVM Landelijk Meetnet Luchtkwaliteit and publish this data using various public standards in particular those related to INSPIRE and Eionet. For this purpose a technical platform, "SOSPilot" has been developed. This website provides links to various elements of the SOSPilot platform.

Clients - AQ Data

Various clients that connect to the Web Services. Click on an image to launch each client.

HeronViewer - Web-based viewer for WMS and WFS - Raw measurments data viewing and searching via WMS and WFS. Both last measured values per component and station and time-series via a timeslider using WMS-Time. Based on Heron Mapping Client.
This app uses the js-sensorweb-client. This is a modern HTML5/responsive design app and the most user-friendly. It uses the REST API from the SOS service (see Web Services below) directly.
This app uses the 52North sos-js client library. It shows a map with stations. One can select an offering (Station) and visualize data via a plot or table for a chemical component time frame.
52North SOS WebClient.
Note that not all stations will output all components. SO2, benzeen (Utrecht-Erzeijstraat), and PM2.5 (Hilvarenbeek) are very rare. Most stations will provide observations for NO2.

Clients - Weather Data

In addition to AQ data, weather data is also exercised in the project.

Report from the Geonovum Davis Weather Station
Python weewx weather software on a Raspberry Pi collects weather data from the Davis Vantage Pro2 Weather Station on the Geonovum-building rooftop. weewx generates this report every 5 mins. The weather data is also published to WMS-Time/WFS and SOS services similar to RIVM AQ data.
